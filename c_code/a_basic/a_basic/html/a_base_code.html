내용을 작성하는 요소로서, 단락(paragraph)이라는 의미를 가지고 있다.

웹문서에서는 띄어쓰기를 여러번하거나, 엔터를 이용하여 줄바꿈처리하는것은 띄어쓰기 한칸으로 인지하며,
만약, 줄바꿈을 처리해서 구분하고자 한다면 <br />을 작성해야한다. 

직접 띄어쓰기나 줄바꿈을 원하는대로 처리하고자 한다면 pre요소를 사용하면 되나,

    실제로는 매우 불편하므로 코드의 내용을 표현하는 형태가 아닌 경우엔 사용률이 매우 저조함

내용을 작성할땐, 때로 강조를 하고자 할 때도 있다.
굵은 글씨를 사용하거나, 기울이거나, 밑줄을 표기하거나, 취소선을 넣거나...
그러나 위 내용은 모두 강조의 의미가 아니다

웹문서에서 강조의 의미는 말 그대로 그 뜻을 전달 가능해야 한다.
매우 중요 - 즉, 위험과 유사한 중요강조를 뜻하는 코드
중요한 의미이지만, 강요보다는 권유의 성격을 띄고있는 코드
강조 처리되는 기능으로 재확인, 기억의 의미
이미 처리가 되어 현재 내용을 더이상 사용할 수 없다

i요소는 감정표현이라는 의미를 가지므로 주로, 아이콘 또는 이모티콘의 용도로 사용
s요소(strike)는 html5에서 완전히 삭제된 요소이므로 더이상 쓸 수 없다.

내용상 용어를 사용할 경우, 요약형태를 사용하는 경우가 많다.
